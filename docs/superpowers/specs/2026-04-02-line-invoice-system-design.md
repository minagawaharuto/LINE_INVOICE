# LINE請求書自動作成システム 設計書

**作成日：** 2026-04-02
**ステータス：** 承認済み

---

## 1. 概要

ダンサー派遣会社が、クライアント企業からLINE公式アカウント経由で勤務・案件情報を受け取り、月次のGoogleスプレッドシート（外注連絡表）を自動生成・追記するシステム。

---

## 2. ビジネス要件

| 項目 | 内容 |
|------|------|
| 入力者 | クライアント企業（ダンサーを発注した企業） |
| 入力チャネル | LINE公式アカウント（LIFFフォーム） |
| 入力項目 | 日程・案件名・名前・数量・詳細（任意） |
| 出力 | 月次Googleスプレッドシート（Google Drive保存） |
| ファイル名形式 | `【ダンサー】令和○年○月分 外注連絡表` |
| 管理方法 | 管理者はGoogle Sheetsを直接確認・編集 |

---

## 3. アーキテクチャ

```
【クライアント企業】
  LIFFフォーム（HTML/JS）
    │ HTTPS POST
    ▼
【Google Apps Script（GAS）】
  ├── doPost() - Webhookエンドポイント
  ├── parseFormData() - フォームデータパーサー
  ├── lookupMaster() - マスタ参照（案件名→単価）
  ├── appendToSheet() - 月次スプレッドシートへ追記
  ├── createMonthlySheet() - 月初トリガー（毎月1日）
  └── replyToLine() - LINE返信
    │
    ▼
【Google Drive】
  ├── 月次スプレッドシート
  │     ├── 入力表シート
  │     └── 外注連絡票シート（数式で自動集計）
  └── マスタスプレッドシート
        ├── 案件マスタシート（案件名・単価）
        └── ダンサーマスタシート（名前・時給・交通費・口座情報）
```

| 要素 | 技術 |
|------|------|
| フロントエンド | LIFF SDK + HTML / CSS / JavaScript |

## 5. LINEフロントエンド（LIFF）

### 5-1. リッチメニュー構成

- トーク画面下部に常時表示
- ボタン：「データ入力」→ LIFFフォームを開く

### 5-2. LIFFフォーム仕様

```
┌─────────────────────────────┐
│  案件入力フォーム              │
├─────────────────────────────┤
│ 日程     [カレンダーピッカー  ] │
│ 案件名   [ドロップダウン      ] │
│ 名前     [ドロップダウン      ] │
│ 数量     [数値入力            ] │
│ 詳細     [テキスト入力（任意） ] │
├─────────────────────────────┤
│ ＋ 行を追加                   │
├─────────────────────────────┤
│         [ 送 信 ]             │
└─────────────────────────────┘
```

- **案件名・名前**：マスタスプレッドシートからドロップダウン生成
- **日程**：カレンダーピッカー（デフォルト：本日）
- **行を追加**：同一送信で最大10件まで入力可能
- **送信後**：LINEトーク画面に完了メッセージが届く

---

## 6. GASバックエンド

### 6-1. エンドポイント

| 関数 | 役割 |
|------|------|
| `doPost(e)` | LIFFからのPOSTを受信・処理のエントリーポイント |
| `parseFormData(data)` | フォームデータをパースし配列に変換 |
| `lookupMaster(jobName)` | 案件マスタから単価を取得 |
| `getOrCreateMonthlySheet()` | 当月のスプレッドシートを取得（なければ作成） |
| `appendRows(sheet, rows)` | 入力表シートに行を追記 |
| `replyToLine(replyToken, message)` | LINE Messaging APIで返信 |

### 6-2. 月初トリガー

- 毎月1日0時にGASのタイムトリガーで `createMonthlySheet()` を実行
- Google Driveの指定フォルダに新規スプレッドシートを作成
- テンプレートスプレッドシートをコピーして初期化

### 6-3. スプレッドシート処理フロー

```
LIFFフォーム送信
  → GAS doPost()
  → 当月スプレッドシートを取得（なければ作成）
  → 各行について：
      - 案件マスタから単価を参照
      - 合計金額 = 数量 × 単価
      - 入力表シートの末尾に追記
  → LINE返信：「✅ ○件追加しました（YYYY/MM/DD）」
```

## 7. Googleスプレッドシート構成

### 7-1. 月次スプレッドシート

**ファイル名：** `【ダンサー】令和8年4月分 外注連絡表`（和暦自動計算）

#### 入力表シート

| 列 | 項目 | 備考 |
|----|------|------|
| A | 日程 | YYYY/MM/DD形式 |
| B | 案件名 | テキスト |
| C | 詳細 | テキスト（任意） |
| D | 名前 | ダンサー芸名 |
| E | 数量 | 数値 |
| F | 単価 | マスタ参照値 |
| G | 合計金額 | =E×F（数式） |
| H | 登録日時 | GASが自動記録 |

#### 外注連絡票シート

既存の `【ダンサー】編集用.xlsx` の外注連絡票フォーマットに準拠。入力表シートの集計を数式で自動反映。

| 列 | 項目 |
|----|------|
| 在籍 | THP Dancers 等 |
| コード | ダンサーコード |
| 芸名 / 氏名 | マスタ参照 |
| 案件報酬（出演費） | 入力表から集計 |
| 案件報酬（インセンティブ） | 入力表から集計 |
| 交通費 | マスタ参照 × 出勤日数 |
| 課税支払 | 自動計算 |
| 所得税（10.21%） | 自動計算 |
| 差引支払額 | 自動計算 |
| 振込金額 | 自動計算 |
| 振込先口座 / 口座番号 / 口座名義 | マスタ参照 |

### 7-2. マスタスプレッドシート（別ファイル）

#### 案件マスタシート

| 案件名 | 現場コード | 単価 | 項目区分 |
|--------|-----------|------|---------|
| OWL TIP | OWL | 50 | TIP |
| PCDL GUEST | PCDL | 1000 | ゲスト |
| ... | ... | ... | ... |

#### ダンサーマスタシート

| 芸名 | 本名 | コード | 時給 | 交通費 | 在籍 | 金融機関 | 口座番号 | 口座名義 | 住所 | 連絡先 |
|------|------|--------|------|--------|------|---------|---------|---------|------|--------|

| 設定項目 | 内容 |
|---------|------|
| チャネル種別 | Messaging API |
| Webhook URL | GAS WebアプリのデプロイURL |

| リッチメニュー | 「データ入力」ボタン1つ（最小構成） |

---

## 9. データフロー

```
1. クライアント企業がリッチメニュー「データ入力」をタップ
2. LIFFフォームが開く
3. 日程・案件名（ドロップダウン）・名前（ドロップダウン）・数量・詳細を入力
4. 「行を追加」で複数件入力（任意）
5. 「送信」をタップ
6. GAS doPost() が受信
7. 案件マスタから単価を参照・合計金額を計算
8. 当月スプレッドシートの入力表に追記
9. LINEトーク画面に「✅ 3件追加しました（2026/04/01）」と返信
```

---

## 10. 月次スプレッドシート自動生成

```
毎月1日 0:00（GASタイムトリガー）
  → テンプレートスプレッドシートをコピー
  → ファイル名を「【ダンサー】令和○年○月分 外注連絡表」に設定（和暦計算）
  → 指定のGoogle DriveフォルダにファイルをMOVE
  → ダンサーマスタから外注連絡票の氏名・口座情報を転記
  → 前月スプレッドシートへのリンクをコメントに残す
```

---

## 11. 制約・注意事項

- GASのWebアプリURLは固定（デプロイのたびに変わらない設定にする）
- LIFFのエンドポイントURLはLINE Developersコンソールで事前登録が必要
- マスタスプレッドシートのID（ファイルID）はGASのスクリプトプロパティで管理
- LINE Messaging APIの送信上限（月1000通無料）に注意
- GASの実行時間制限：1回あたり6分（通常の追記処理では問題なし）
